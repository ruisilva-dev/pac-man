## ⚙️ 2. Repository Management & Integration Workflow

To maintain a pristine, atomic Git history for final review, the team operates via a strict dual-repository system:
* **Feature Branching / Testing:** Shared workspace where individual sub-modules, sprites, and experiments are pushed iteratively for collaborative cross-testing.
* **Official Main Branch / Integration:** Managed exclusively by **ruisilva**. Responsible for auditing code blocks against PEP 8/Flake8/Mypy standards, filtering structural deltas, resolving dependencies, and compiling stable feature milestones into logically organized, atomic Git commits on behalf of the team.
